# NarrRail WebEditor UI Style Guide

本规范用于统一 WebEditor 的按钮、弹窗、面板与交互细节，默认在后续迭代中遵循。

## 1. 总体原则

1. 低干扰优先：控件不抢内容焦点。
2. 内容优先：文本/节点内容层级高于控制按钮。
3. 一致性优先：同类行为在不同页面保持一致视觉和交互。
4. 渐进显现：次要/危险操作默认弱化，仅在 hover/focus 时增强。

## 2. 按钮设计原则

### 2.1 默认按钮风格

- 优先使用透明底、无边框或极弱边框。
- 避免高饱和、大块实心按钮长期占据视线。
- 图标按钮优先于“图标+文字”按钮（除关键主流程）。

### 2.2 强调层级

按钮按视觉优先级分层：

1. 主内容交互（最高）
2. 当前上下文必要操作
3. 辅助操作（如全屏、排序）
4. 关闭/收起等结构性操作（最低，默认弱化）

### 2.3 危险操作

- 删除等危险行为默认不常驻高亮。
- 仅在 hover 到目标行、目标卡片后显示或增强。
- 避免大面积红色背景常驻。

## 3. 弹窗规范

### 3.1 布局与尺寸

- 弹窗应以“画布内比例窗口”为默认，不使用过高全屏占比。
- 推荐使用宽高比（如 16:9）+ 最大高度限制，避免纵向压迫。

### 3.2 关闭行为

- 默认关闭方式：
  - `Esc`
  - 点击弹窗外遮罩区域
- 禁止点击弹窗内容区误关闭。

### 3.3 结构简化

- 能通过快捷键/外部点击完成的流程，不强制保留底部“完成/确认”区。
- 不必要的分隔线、重复按钮应移除。

## 4. 输入区规范（多行编辑）

- 默认以单行高度起步（可容纳一行文字）。
- 内容换行后自动伸展，避免初始占高过大。
- 行操作（拖拽、删除）要弱化视觉，不压过文本内容。

## 5. 面板与上下文关系

- 当开启全局编辑弹窗时，相关侧边属性面板应自动收起，避免“弹窗嵌套在面板中”的错觉。
- 编辑焦点应保持在弹窗上下文，防止输入导致弹窗状态被重置。

## 6. 交互一致性清单（提交前自查）

1. 是否存在高干扰按钮长期高亮？
2. 非主操作是否已弱化？
3. 点击内容区是否会误触关闭？
4. 输入时是否会导致弹窗异常关闭/重置？
5. 行高、间距、边框是否保持轻量统一？
6. 同类按钮（关闭/拖拽/删除）在不同页面是否风格一致？

## 7. 变更策略

后续 UI 改动默认遵循本规范。若出现冲突：

- 以“内容可读性 + 低干扰 + 一致性”为最终裁决标准。
- 在 PR/提交说明中标注“偏离规范”的原因与取舍。

# NarrRail 主介绍页面内容与排版规范（Draft v0.1）

> 适用范围：`NarrRail.WebEditor` 新增“主介绍页面”功能
> 
> 目标：在用户进入编辑器之前，用一个清晰、可信、低学习成本的页面说明 **Web 编辑器能力**、**UE 插件能力**、以及二者如何协同工作。

---

## 1. 文档目标

本规范用于指导主介绍页面的设计与实现，重点定义：

1. 介绍页面需要传达的关键信息；
2. Web 编辑器功能要点（面向创作人员）；
3. UE 插件功能要点（面向集成人员/技术人员）；
4. 二者协作关系与工作流；
5. 页面排版风格、视觉层级与交互节奏；
6. 可执行的验收标准（DoD）。

---

## 2. 页面定位

### 2.1 产品定位

主介绍页面是 NarrRail 的“入口说明页”，不是营销官网，不追求花哨动效，优先：

- 解释清楚“这是做什么的”；
- 告诉用户“我下一步该做什么”；
- 降低首次使用门槛；
- 强化 WebEditor ↔ UE 插件的一体化认知。

### 2.2 用户类型

1. **编剧/叙事策划**：关心剧情编辑效率、导入导出与校验反馈；
2. **客户端/技术美术/关卡技术**：关心 UE 运行时执行、调试、集成路径；
3. **团队负责人**：关心流程闭环、版本协作、可维护性。

---

## 3. 入口与导航要求

### 3.1 当前阶段入口要求（你提出的约束）

在主页（Library/Home）中，将“介绍页入口”先放在：

- **登录按钮旁边**
- **切换主题模式按钮旁边**

建议按钮命名：

- 中文：`产品介绍`
- 英文：`Overview`

建议图标：`info` / `dashboard` / `overview`

### 3.2 跳转行为

- 点击入口后进入介绍页面（单独视图，不覆盖已有编辑状态）；
- 支持返回主页；
- 保留当前主题（深色/浅色）一致性。

---

## 4. 页面信息架构（IA）

介绍页面建议采用以下分区（从上到下）：

1. **Hero 区（价值主张）**
2. **Web 编辑器功能要点**
3. **UE 插件功能要点**
4. **协作流程（Web → YAML → UE Runtime）**
5. **适用场景 / 角色分工**
6. **快速开始 CTA（行动区）**

---

## 5. Web 编辑器功能要点（内容草案）

建议以“能力卡片”方式展示，避免大段说明文字。

### 5.1 图形化剧情编辑

- 节点化创作（Dialogue / MultiDialogue / Choice / Jump / SetVariable / EmitEvent / End）
- 拖拽连线、右键建点、自动排布
- Choice 分支句柄映射与可视化

### 5.2 内容编辑效率

- 对话与多行对话的弹窗编辑
- 长文本可读显示（非截断）
- Enter 快捷新增、拖拽排序（多行/选项）

### 5.3 数据与质量保障

- YAML 导入导出（含兼容格式处理）
- 实时校验 + 手动校验
- 本地自动保存（localStorage）

### 5.4 阅读与审校

- 阅读模式（Read Mode）用于文本校对
- 主分支顺序化阅读，聚焦“可读性”

---

## 6. UE 插件功能要点（内容草案）

该部分强调“运行时与引擎侧能力”，避免与 Web 编辑器重复。

### 6.1 Runtime 执行核心（C++）

- 剧情会话执行：`Start / Next / Choose / Pause / Resume / Stop`
- Choice 模式：`SinglePass` / `ExhaustiveUntilComplete`
- 条件求值与动作执行（变量/事件）

### 6.2 数据资产与校验

- Story Asset 数据模型（节点、边、变量）
- 导入后结构校验（断链、重复 ID、无效映射）
- 版本迁移入口（Schema 兼容演进）

### 6.3 调试与集成

- Blueprint API 暴露用于业务层调用
- 会话事件广播（节点进入、选择就绪、选择结果）
- HostProject 中编译、PIE 验证与问题定位

---

## 7. Web 编辑器与 UE 插件协作关系

### 7.1 一句话关系

- **WebEditor 负责“创作与结构正确性”**
- **UE 插件负责“运行时执行与引擎集成”**

### 7.2 标准工作流

1. 在 WebEditor 完成剧情结构编辑与校验；
2. 导出标准 YAML；
3. 在 UE 中导入为 Story Asset；
4. 在 PIE/游戏逻辑中运行会话并调试；
5. 发现问题回流到 WebEditor 修改，再导出。

### 7.3 数据契约要点

- Script Schema 保持版本兼容；
- Choice/Jump 等字段命名与结构在两端对齐；
- 导入器对历史格式做兼容降级；
- 错误提示需可追踪到节点/字段。

## 8. 介绍页排版与视觉风格规范

结合现有 WebEditor 视觉语言（glassmorphism + 高可读）定义：

### 8.1 风格关键词

- 清晰、可信、克制
- 技术产品感，但不冰冷
- 内容优先、低干扰

### 8.2 栅格与节奏

- 页面内容宽度：`min(1200px, 92vw)`
- 分区纵向间距：`56~80px`
- 区内卡片间距：`16~24px`
- 标题层级：`H1 > H2 > CardTitle`

### 8.3 组件建议

- Hero：标题 + 副标题 + 两个 CTA（开始使用 / 查看文档）
- 能力区：3 列卡片（桌面），1 列（移动端）
- 协作关系：流程图/步骤条（Web → YAML → UE）
- CTA 区：醒目但不过度高饱和

### 8.4 文案规范

- 每条能力描述控制在 1~2 行；
- 先结果后机制（先说“能做什么”，再说“怎么做到”）；
- 避免术语堆叠，必要术语配简短注释。

### 8.5 深浅主题适配

- 保持现有主题 token 体系；
- 深色模式下保证卡片边界与正文对比度；
- 禁止依赖单一颜色传达状态。

### 8.6 动效原则

- 微动效即可（hover / fade / slight lift）；
- 禁止大幅持续动画分散注意力；
- 首屏加载优先速度与稳定性。

## 10. 实施建议（前端层）

### 10.1 路由/状态建议

可在当前 `currentView` 基础上新增：

- `currentView = 'overview'`

并在主页工具区域增加入口按钮，与登录/主题切换并列。

### 10.2 组件建议

新增组件：

- `src/components/OverviewPage.vue`

职责：

- 只读展示介绍内容；
- 统一承接主题样式；
- 提供返回主页与开始创作的 CTA。

---

## 11. 验收标准（DoD）

1. 首页存在“介绍页入口”，位置在登录与主题切换按钮附近；
2. 点击可进入介绍页并可返回；
3. 页面完整覆盖以下四类内容：
   - Web 编辑器功能要点
   - UE 插件功能要点
   - 协作关系
   - 排版风格一致性
4. 深色/浅色主题显示正确，无可读性问题；
5. 移动端基本可用（卡片自动单列，文字不溢出）。

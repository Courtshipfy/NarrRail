# NarrRailEditor 阅读模式（Read Mode）设计说明

> 状态：已落地（v1）/ 持续迭代
> 更新：2026-05-22

---

## 1. 背景与目标

> 注：本文件最初为“阅读模式”规划稿。当前已演进为“预览模式（Preview Mode）”，即按运行语义推进的审校视图。

当前 NarrRailEditor 以「节点图编辑」为中心，适合搭建和修改结构；但在审阅剧情、校对台词节奏时，节点视图的信息密度偏高。

**阅读模式（Read Mode）** 的目标是提供一个“只读、精简、顺序化”的阅读体验：

- 不进行节点编辑；
- 不展示复杂连线关系；
- 按剧情执行顺序逐行阅读；
- 以统一文本格式呈现：`人物：对话内容`。

---

## 2. 产品定义

### 2.1 模式定位

阅读模式是编辑器内的一个**状态切换**，不是新页面。

- 与当前编辑模式互斥；
- 切回编辑模式时，原图状态（视口/选中）可恢复。

### 2.2 入口位置

在顶部工具栏增加阅读模式开关按钮（Toggle）：

- 文案建议：`阅读模式` / `Read Mode`
- 图标建议：`article` 或 `menu_book`
- 状态：`开启 / 关闭`

### 2.3 核心输出形态

开启后以“文章阅读”形式显示文本列表：

- 一行一条可读单元；
- 基础格式：`人物：对话内容`；
- 说话人为空时显示：`旁白：对话内容`；
- 多行 `Dialogue` 按原行顺序输出。

---

## 3. 读取/执行规则（v1）

> v1 已实现“可推进、可分支、可结束”的运行预览核心闭环。

### 3.1 节点类型处理

- `Dialogue`：按台词行输出；单行输出 1 行，多行按原行顺序输出
- `Choice`：
  - 显示可选项按钮，用户点击后进入对应分支
  - 支持 `ExhaustiveUntilComplete`：记录已选项、分支结束后返回 Choice，直至耗尽并走 `choice-complete`
- `Jump`：写入预览时间线（`[跳转]`）并推进到目标
- `SetVariable` / `EmitEvent`：写入预览时间线（`[变量]` / `[事件]`）
- `End`：写入 `[结束]` 后结束会话；结束态点击不再重复触发

### 3.2 顺序来源

顺序基于运行时语义的“可执行路径”而非画布坐标：

1. 从 `entryNodeId` 起始；
2. 沿边推进（条件求值后选择可用边）；
3. 在 `await-choice` 状态等待用户选择；
4. 无可用后续边时结束（或在穷举模式中返回上层 Choice）。

---

## 4. 交互与 UI 要求

### 4.1 模式切换行为

开启阅读模式时：

- 主画布切换为阅读视图容器；
- 节点编辑交互关闭（拖拽、连线、右键建点等）；
- 属性面板隐藏或降级为只读摘要（v1 建议隐藏）。

关闭阅读模式时：

- 返回图编辑画布；
- 恢复之前编辑上下文。

### 4.2 预览视图样式（现状）

- 版式偏“文档流”：居中列宽、舒适行距；
- 对白行渲染：`人物：对话内容`；
- 元信息行渲染：`[选择] / [变量] / [事件] / [跳转] / [结束]`；
- 元标签颜色与节点类型配色对齐；
- 支持深浅主题适配；
- 预览区域默认禁选中文本（避免连续点击产生选区）。

### 4.3 快捷操作（建议）

- `R`：切换阅读模式（可选）
- `Esc`：退出阅读模式（可选）

## 5. 数据与代码设计建议

### 5.1 状态层

在 `App.vue` 增加全局状态：

- `isReadModeEnabled: boolean`
- `readModeLines: ReadLine[]`

`ReadLine` 建议结构：

- `nodeId: string`
- `lineIndex?: number`（多行 Dialogue 行索引）
- `speaker: string`
- `text: string`
- `kind: 'dialogue' | 'choice-hint'`

### 5.2 组件层

新增组件建议：

- `src/components/ReadModePanel.vue`

职责：

- 接收 `readModeLines`
- 负责渲染阅读列表
- 处理基础滚动/复制/主题样式

### 5.3 解析层

新增工具建议：

- `src/utils/read-mode-builder.js`

职责：

- 输入：`nodes, edges, storyMeta`
- 输出：`ReadLine[]`
- 封装节点遍历与文本提取规则

---

## 6. 与迭代同步策略

阅读模式是“长期同步功能”，建议每次新增节点能力时同步更新：

1. 在该节点功能 PR 中增加阅读模式处理说明；
2. 若该节点影响阅读顺序/文本语义，更新 `read-mode-builder` 规则；
3. 更新本文档“节点类型处理”章节。

---

## 7. 分阶段实施计划

### 已完成（v1）

- 工具栏模式切换
- 运行语义推进（含 Choice 选择）
- `Dialogue / Choice / Jump / SetVariable / EmitEvent / End` 支持
- 结束态保护（不重复触发）
- `ExhaustiveUntilComplete` 穷举分支支持

### 待迭代（v1.x / v2）

- 技术节点显示开关（详细/精简）
- 行定位回跳（点击预览行高亮节点）
- 预览搜索、统计、纯文本导出
- 与 Runtime C++ 执行结果一致性回归测试

---

## 8. 验收标准（DoD）

- 能在工具栏稳定切换阅读模式；
- 阅读模式下无编辑副作用；
- 文本顺序可复现、可解释；
- 单行 / 多行 `Dialogue` 文本格式正确输出；
- 深浅主题下可读性达标。

---

## 9. 风险与注意事项

- 分支剧情天然存在“多路径”歧义，v1 需明确“只看主路径”；
- 不应把阅读模式做成“另一个编辑器”（坚持只读）；
- 后续节点类型扩展时，务必同步更新本说明。
